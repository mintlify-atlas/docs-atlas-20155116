title: Quickstart
description: Get a working TaskForge API call in under 5 minutes.
---

This guide takes you from zero to your first authenticated API call against a locally running TaskForge instance.

## Prerequisites

- **Python 3.11+** — TaskForge requires Python 3.11 or newer.
- **git** — to clone the repository.
- **Docker** _(optional)_ — if you prefer a containerised setup, see the [Docker guide](/guides/docker).

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/YamiDarknezz/task-forge-api.git
    cd task-forge-api
    ```
  </Step>

  <Step title="Create a virtual environment and install dependencies">
    <CodeGroup>
      ```bash Linux / macOS
      python3 -m venv .venv
      source .venv/bin/activate
      pip install -r requirements.txt
      ```

      ```bash Windows
      python -m venv .venv
      .venv\Scripts\activate
      pip install -r requirements.txt
      ```
    </CodeGroup>

    <Tip>
      Key packages installed include Flask 3.1.2, Flask-JWT-Extended 4.7.1, Flask-SQLAlchemy 3.1.1, and Flask-Limiter 4.0.0. See `requirements.txt` for the full list.
    </Tip>
  </Step>

  <Step title="Configure environment variables">
    Copy the example env file and edit the required values:

    <CodeGroup>
      ```bash Linux / macOS
      cp .env.example .env
      ```

      ```bash Windows
      copy .env.example .env
      ```
    </CodeGroup>

    Open `.env` and set at minimum:

    ```bash .env
    FLASK_APP=run.py
    FLASK_ENV=development
    SECRET_KEY=tu-clave-secreta-aqui-cambiar-en-produccion
    JWT_SECRET_KEY=tu-clave-jwt-secreta-aqui-cambiar-en-produccion

    # JWT token lifetimes (seconds)
    JWT_ACCESS_TOKEN_EXPIRES=3600
    JWT_REFRESH_TOKEN_EXPIRES=2592000

    # Rate limiting
    RATELIMIT_ENABLED=true
    RATELIMIT_STORAGE_URL=memory://
    RATELIMIT_DEFAULT=200 per day;50 per hour
    ```

    <Note>
      For local development you do **not** need to set the `AZURE_SQL_*` variables. When they are absent, the app automatically uses SQLite (`taskforge.db` in the project root).
    </Note>

    <Warning>
      Replace `SECRET_KEY` and `JWT_SECRET_KEY` with strong random values before exposing this server to any network. Never commit real secret values to source control.
    </Warning>
  </Step>

  <Step title="Initialise the database">
    Run the built-in CLI command to create all tables:

    ```bash
    flask init-db
    ```

    You should see:

    ```
    Base de datos inicializada con exito!
    ```

    <Note>
      The `init-db` command calls `db.create_all()` and is defined in `run.py`. It creates a local `instance/taskforge.db` SQLite file when no Azure SQL credentials are present.
    </Note>
  </Step>

  <Step title="Start the server">
    ```bash
    python run.py
    ```

    The server listens on port `5000` by default (controlled by the `PORT` environment variable). You should see Flask's development server output:

    ```
     * Running on http://0.0.0.0:5000
    ```

    The interactive Swagger UI is available at [http://localhost:5000/api/docs](http://localhost:5000/api/docs).
  </Step>

  <Step title="Make your first API calls">
    With the server running, make three calls in sequence: register a user, log in to get tokens, then create a task.

    ### 1. Register a user

    ```bash
    curl -X POST http://localhost:5000/api/auth/register \
      -H "Content-Type: application/json" \
      -d '{
        "username": "johndoe",
        "email": "john@example.com",
        "password": "SecurePass123",
        "first_name": "John",
        "last_name": "Doe"
      }'
    ```

    **Response `201`:**

    ```json
    {
      "success": true,
      "message": "Usuario registrado con exito",
      "data": {
        "id": 1,
        "username": "johndoe",
        "email": "john@example.com",
        "first_name": "John",
        "last_name": "Doe",
        "full_name": "John Doe",
        "is_active": true,
        "role": { "id": 2, "name": "user" }
      }
    }
    ```

    <Note>
      Registration is rate-limited to **5 requests per hour** per IP address.
    </Note>

    ### 2. Log in and get tokens

    ```bash
    curl -X POST http://localhost:5000/api/auth/login \
      -H "Content-Type: application/json" \
      -d '{
        "email": "john@example.com",
        "password": "SecurePass123"
      }'
    ```

    **Response `200`:**

    ```json
    {
      "success": true,
      "message": "Inicio de sesion exitoso",
      "data": {
        "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "user": {
          "id": 1,
          "username": "johndoe",
          "email": "john@example.com"
        }
      }
    }
    ```

    Save the `access_token` — you need it for every subsequent request.

    ### 3. Create a task

    Replace `<access_token>` with the token from the login response:

    ```bash
    curl -X POST http://localhost:5000/api/tasks \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer <access_token>" \
      -d '{
        "title": "Complete project documentation",
        "description": "Write comprehensive API documentation",
        "priority": "high",
        "due_date": "2024-12-31T23:59:59"
      }'
    ```

    **Response `201`:**

    ```json
    {
      "success": true,
      "message": "Tarea creada con exito",
      "data": {
        "id": 1,
        "title": "Complete project documentation",
        "description": "Write comprehensive API documentation",
        "status": "pending",
        "priority": "high",
        "due_date": "2024-12-31T23:59:59",
        "tags": []
      }
    }
    ```
  </Step>
</Steps>
