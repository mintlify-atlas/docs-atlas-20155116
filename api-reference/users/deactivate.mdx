---
title: Deactivate a user
description: Disable a user account so the user can no longer authenticate.
---

Sets `is_active` to `false` for the specified user. A deactivated user cannot obtain new tokens or make authenticated API requests. Their data and tasks are preserved.

<Warning>
  This endpoint requires **admin** privileges. Requests from non-admin users return `403 Forbidden`.
</Warning>

<Warning>
  Admin accounts cannot be deactivated. Attempting to deactivate an admin user returns `403 Forbidden`.
</Warning>

## Authentication

Requires a valid Bearer token from an admin account in the `Authorization` header.

## Path parameters

<ParamField path="user_id" type="number" required>
  The unique integer ID of the user to deactivate.
</ParamField>

## Response fields

<ResponseField name="success" type="boolean" required>
  `true` when the deactivation succeeds.
</ResponseField>

<ResponseField name="message" type="string" required>
  Confirmation message.
</ResponseField>

<ResponseField name="data" type="object" required>
  The updated user object with `is_active: false`.
  <Expandable title="properties" defaultOpen={true}>
    <ResponseField name="id" type="number" required>
      Unique identifier for the user.
    </ResponseField>
    <ResponseField name="username" type="string" required>
      The user's unique username.
    </ResponseField>
    <ResponseField name="email" type="string" required>
      The user's email address.
    </ResponseField>
    <ResponseField name="first_name" type="string">
      User's first name. `null` if not set.
    </ResponseField>
    <ResponseField name="last_name" type="string">
      User's last name. `null` if not set.
    </ResponseField>
    <ResponseField name="full_name" type="string" required>
      Computed display name.
    </ResponseField>
    <ResponseField name="is_active" type="boolean" required>
      `false` after this operation completes.
    </ResponseField>
    <ResponseField name="role" type="object" required>
      The user's assigned role.
      <Expandable title="role properties">
        <ResponseField name="id" type="number" required>
          Role ID.
        </ResponseField>
        <ResponseField name="name" type="string" required>
          Role name: `admin` or `user`.
        </ResponseField>
        <ResponseField name="description" type="string">
          Role description.
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="created_at" type="string" required>
      ISO 8601 timestamp of account creation.
    </ResponseField>
    <ResponseField name="updated_at" type="string" required>
      ISO 8601 timestamp of the most recent update.
    </ResponseField>
  </Expandable>
</ResponseField>

## Errors

| Status | Description |
| ------ | ----------- |
| `401`  | Missing or invalid Bearer token. |
| `403`  | Authenticated user is not an admin, or the target user is an admin. |
| `404`  | No user exists with the given `user_id`. |

## Examples

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api.taskforge.example.com/api/users/2/deactivate \
  --header 'Authorization: Bearer <admin-token>'
```

```python Python
import requests

response = requests.post(
    "https://api.taskforge.example.com/api/users/2/deactivate",
    headers={"Authorization": "Bearer <admin-token>"},
)
print(response.json())
```

</CodeGroup>
