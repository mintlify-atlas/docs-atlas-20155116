---
title: Create a tag
description: Create a new tag for categorizing tasks.
---

Creates a new tag. Any authenticated user can create tags. The tag name must be unique across the system.

## Authentication

Requires a valid Bearer token in the `Authorization` header.

<Info>
  This endpoint is rate-limited to **20 requests per hour** per authenticated user.
</Info>

## Request body

<ParamField body="name" type="string" required>
  Display name for the tag. Must be between 1 and 50 characters. Must be unique.
</ParamField>

<ParamField body="color" type="string" default="#808080">
  Hex color code to represent the tag visually (e.g. `#FF5733`). Must match the pattern `^#[0-9A-Fa-f]{6}$`. Defaults to `#808080` (gray) if omitted.
</ParamField>

<ParamField body="description" type="string">
  Optional human-readable description of the tag. Maximum 255 characters.
</ParamField>

## Response fields

<ResponseField name="success" type="boolean" required>
  `true` when the tag is created successfully.
</ResponseField>

<ResponseField name="message" type="string" required>
  Confirmation message.
</ResponseField>

<ResponseField name="data" type="object" required>
  The newly created tag object.
  <Expandable title="properties" defaultOpen={true}>
    <ResponseField name="id" type="number" required>
      Unique identifier assigned to the new tag.
    </ResponseField>
    <ResponseField name="name" type="string" required>
      Tag name.
    </ResponseField>
    <ResponseField name="color" type="string" required>
      Hex color code.
    </ResponseField>
    <ResponseField name="description" type="string">
      Tag description. `null` if not provided.
    </ResponseField>
    <ResponseField name="task_count" type="number" required>
      Always `0` for a newly created tag.
    </ResponseField>
    <ResponseField name="created_at" type="string" required>
      ISO 8601 timestamp of creation.
    </ResponseField>
    <ResponseField name="updated_at" type="string" required>
      ISO 8601 timestamp of last update.
    </ResponseField>
  </Expandable>
</ResponseField>

## Errors

| Status | Description |
| ------ | ----------- |
| `400`  | Validation failed — `name` is missing, too long, or `color` is not a valid hex code. |
| `401`  | Missing or invalid Bearer token. |
| `409`  | A tag with that `name` already exists. |
| `429`  | Rate limit exceeded (20 requests/hour). |

## Examples

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api.taskforge.example.com/api/tags \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "name": "Urgent",
    "color": "#FF5733",
    "description": "High-priority tasks that need immediate attention"
  }'
```

```python Python
import requests

response = requests.post(
    "https://api.taskforge.example.com/api/tags",
    headers={
        "Authorization": "Bearer <token>",
        "Content-Type": "application/json",
    },
    json={
        "name": "Urgent",
        "color": "#FF5733",
        "description": "High-priority tasks that need immediate attention",
    },
)
print(response.json())
```

</CodeGroup>
